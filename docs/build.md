# Build

> *Build preserves what the Mission creates.*

The Build represents the completed artifact of an Adventure.

Unlike the Mission, which focuses on the implementation process, the Build captures the finished result.

It answers a simple question:

> **What now exists because this Adventure was completed?**

---

# Purpose

The Build preserves the outcome of the work.

It allows Future Me—and future explorers—to understand, inspect and reuse what was created without repeating the entire Adventure.

The Build is not a journal.

It is not documentation of the process.

It is the artifact itself.

---

# Relationship to the Adventure

Each part of an Adventure has a distinct responsibility.

```text
Training
    ↓
Develop understanding

Mission
    ↓
Create the artifact

Build
    ↓
Preserve the artifact

Field Notes
    ↓
Preserve the journey

Realizations
    ↓
Preserve transferable wisdom

Campfire
    ↓
Preserve personal growth
```

Every document has one responsibility.

The Build exists only to preserve the completed work.

---

# What Belongs in Build?

The Build should contain the deliverables produced by the Adventure.

Examples include:

- architecture diagrams
- network diagrams
- configuration files
- scripts
- source code
- exported policies
- validation reports
- reference implementations
- screenshots that document the finished system
- deployment documentation

The exact contents depend on the Adventure.

---

# What Does Not Belong in Build?

Avoid placing process documentation inside the Build.

Examples include:

- temporary notes
- troubleshooting logs
- reflections
- unfinished ideas
- experimental work
- implementation history

Those belong elsewhere in the Field Guide.

---

# Design Principles

A good Build is:

- complete;
- understandable;
- reproducible;
- reusable;
- well documented.

Someone should be able to inspect the Build and understand what was created without reading the entire Adventure.

---

# Build as Evidence

The Build is the proof that the Adventure happened.

It demonstrates what was created rather than merely describing it.

The stronger the Build, the less explanation it requires.

---

# The Goal

Every Adventure should leave the world richer than it found it.

The Build is the lasting contribution that makes this visible.

The expedition continues.
