# Systems

## The Question

> **What exists?**

Every engineering problem begins with understanding the system itself.

Before analysing relationships, communication or trust, you must first identify the individual parts that make up the system.

The Systems Realm focuses on those parts.

---

## Purpose

The Systems Realm helps you identify, describe and understand the components of a system.

It asks what exists before asking how those components interact.

A good understanding of the individual parts creates the foundation for every other Realm.

---

## What Belongs Here?

Examples include:

- hardware
- operating systems
- virtual machines
- network devices
- databases
- tables
- services
- applications
- users
- groups
- containers
- cloud resources
- files
- scripts

If it exists as an identifiable part of the system, it belongs here.

---

## What Does Not Belong Here?

The Systems Realm does not explain:

- how components interact;
- how information moves;
- whether something can be trusted;
- whether a solution is successful;
- how something should be documented.

Those questions belong to other Realms.

---

## Typical Questions

- What exists?
- Which components make up the system?
- Which responsibilities does each component have?
- Which parts are essential?
- Which parts are optional?
- Which components are missing?
- Which components should be added?

---

## Typical Artifacts

Examples include:

- system inventories
- architecture diagrams
- infrastructure overviews
- hardware lists
- software inventories
- topology diagrams
- entity lists

---

## Relationships with Other Realms

The Systems Realm identifies the parts.

The other Realms build upon that understanding.

Examples:

- Relationships asks how the parts influence one another.
- Communication asks how information moves between them.
- Trust asks why the system can be relied upon.
- Evaluation asks whether the system fulfils its purpose.

---

## The Goal

A complete understanding of the system begins with understanding what exists.

Everything else builds upon that foundation.
