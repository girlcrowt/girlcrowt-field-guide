## Phase Brief

A plan becomes valuable only when it can be put into practice.

During Training, you decided which tools best satisfy the requirements of your Lab.

Now it is time to install those tools.

The objective is not to collect software.

The objective is to build a toolkit that serves a clear purpose.

Every installation should be intentional.

Every decision should remain traceable.

---

## The Goal

Install the tools you selected during Training.

As you work, compare reality against your Toolkit Decision Record.

If your original plan changes, document why.

Engineering decisions evolve.

Undocumented changes become future mysteries.

---

## Suggested Workflow

### Review

Read your Toolkit Decision Record.

Confirm that every selected tool still satisfies your requirements.

---

### Install

Install each required tool.

Avoid installing software "just in case."

Every tool should have a documented purpose.

---

### Verify

After every installation, confirm:

- The software starts correctly.
- The expected features are available.
- The installed version has been recorded.
- No unexpected issues appeared.

---

### Document

Record:

- installed version;
- installation date;
- relevant configuration choices;
- unexpected observations;
- useful references.

Future You should never need to rediscover this information.

---

## Checkpoint

Before continuing to Phase 03, ask yourself:

- Does every installed tool still satisfy my requirements?
- Have I documented every important decision?
- Is every installation working as expected?
- Can I explain why every installed tool exists?
- Is my toolkit ready to support the Lab?

If not, resolve those questions before continuing.

The expedition continues when the toolkit is ready.
